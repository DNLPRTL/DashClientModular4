# Style And Format Rules

## Writing Style

- Prefer concise technical prose.
- Define DASH and ABR terms once, then use them consistently.
- Separate background, implementation, and evaluation claims.
- Avoid overclaiming beyond the implemented and validated work.
- Use "baseline" only for implemented and accepted controllers; use "candidate" for optional methods.

## Formatting Rules

- Use the official/base UGR/ETSIIT LaTeX template for final formatting.
- Keep tables readable; split wide matrices if needed.
- Use original figures with descriptive captions.
- Put long source cards and detailed specs in annexes if they interrupt the main narrative.

## Language Notes

The final thesis language is TBD. If written in Spanish, translate terms consistently and keep English paper titles unchanged.
