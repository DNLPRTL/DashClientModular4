# Rubric Alignment

| rubric dimension | evidence to prepare | current source |
| --- | --- | --- |
| Problem motivation | DASH/HAS relevance and local streaming traffic context | Field map and local related work |
| Technical rigor | ABR-neutral client, controller contract, testability | Phase 1 docs and baseline matrices |
| Scientific grounding | Source cards and survey taxonomy | `00_field_map/` and `01_baselines/` |
| Implementation quality | Future baseline specs, API mappings, tests | Templates and implementation plan |
| Evaluation discipline | Acceptance matrices and later fake-engine methodology | Baseline acceptance matrix |
| Originality | Original diagrams and paraphrased explanations | Originality policy |
| Limitations | Deferred QoE, replay, AI/RL, GStreamer benchmark caveat | Scope decisions and traceability matrix |

## Key Defense Message

The project prioritizes a reproducible, documented path from literature to controller implementation instead of prematurely adding controllers with unclear scientific traceability.
