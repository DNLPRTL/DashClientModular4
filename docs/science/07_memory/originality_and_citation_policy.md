# Originality And Citation Policy

## Originality Rules

- Write explanations in original words.
- Cite every paper, standard, survey, and practical source that informs a claim.
- Do not copy long passages from papers or standards.
- Do not copy figures from papers unless explicit permission is available.
- Prefer original diagrams drawn from the project architecture and documented methodology.
- Do not implement from raw PDFs; implement only from Markdown source cards and specs.

## Citation Rules

| material | action |
| --- | --- |
| DASH terminology | cite ISO/IEC 23009-1:2022 and Stockhammer 2011 as appropriate |
| ABR taxonomy | cite Bentaleb 2019 and recent reviews |
| Local video traffic motivation | cite Ameigeiras 2012 and Ramos-Munoz 2014 |
| Baseline algorithm | cite primary paper and implementation spec |
| Practical dash.js behavior | cite Spiteri 2019 and clearly label as practical/reference-player context |
| Pensieve/RobustMPC comparison | cite Mao 2017 and explain that Pensieve is not implemented |

## Thesis Transparency

Every baseline section should declare whether the implementation is:

- paper-faithful,
- adapted to the local controller API,
- simplified for testability,
- or deferred.
