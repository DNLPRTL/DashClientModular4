# Defense And Submission Requirements

## Defense Preparation

| artifact | purpose |
| --- | --- |
| Baseline selection table | Explain why the initial set is rate-based, BBA, BOLA, MPC, RobustMPC. |
| Signal matrix | Show that each baseline maps to available or explicitly deferred client signals. |
| Architecture diagram | Show separation among parser, downloader, buffer/media engine, controller, logging, and evaluation. |
| Limitation slide | State deferred QoE, replay/traces, AI/RL, and GStreamer benchmark limits. |

## Submission Hygiene

- Do not submit raw PDFs copied into the repo.
- Do not include logs, CSVs, local configs, caches, or media files.
- Ensure bibliography entries match cited text.
- Ensure every figure is original or permission-cleared.
- Ensure annexes do not expose raw copyrighted source material.
